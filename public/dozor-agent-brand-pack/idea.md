# Dozor branding idea

## Core idea
Dozor is an observability product with a night watch metaphor.
The brand should feel calm, vigilant, technical, and precise. The tone is not security-company hard, and not fantasy. It is a modern developer tool that sees what breaks in the dark.

The system uses two marks:

1. **Dozor** - the master brand, represented by **Lantern D**.
2. **Dozor Agent** - the supporting technical mark, represented by **Beacon Tower**.

This creates a clear split of roles:
- the product is the central watch and insight layer;
- the agent is the field node that watches locally and sends signals back.

## Brand architecture

### Master brand: Dozor
Use Lantern D for:
- app logo
- favicon
- dashboard top-left mark
- website hero
- public docs
- login and onboarding
- marketing

### Secondary mark: Dozor Agent
Use Beacon Tower for:
- package page
- installer docs
- agent setup screens
- connection status badges
- onboarding step illustrations
- source cards and host cards

Do not let the agent mark replace the main Dozor mark in public-facing surfaces.

## Visual meaning

### Lantern D
A geometric D that contains a beam of light and a target point.
Meaning:
- observability
- visibility in darkness
- anomaly detection
- calm focus
- technical precision

### Beacon Tower
A watchtower or beacon sending a signal.
Meaning:
- sentinel
- edge node
- agent
- source heartbeat
- constant watch

## Tone and mood
- dark but not grim
- technical but not cold
- vigilant but not paranoid
- premium B2B SaaS
- night-mode native

Think: midnight operations room, precise light, a quiet signal, strong trust.

## Color system

### Core palette
- Midnight: `#081226`
- Graphite: `#0F172A`
- Slate: `#334155`
- Cyan: `#22D3EE`
- Sky: `#7DD3FC`
- Amber: `#F59E0B`
- Rose: `#FB7185`
- Mist: `#E2E8F0`

### Usage
- Midnight and graphite for base surfaces
- Cyan and sky for active monitoring accents
- Amber for warning
- Rose for critical
- Mist for text on dark backgrounds

## Typography direction
Use a modern grotesk / UI sans.
Recommended options:
- Inter
- Geist
- Satoshi
- Manrope

Wordmark should be clean, spaced slightly open, and never decorative.

## Slogans
Primary product line:
- **Observability that keeps watch**

Alternative Dozor lines:
- See what breaks in the dark
- Keep watch over every request
- The calm watch over production

Agent line:
- **The node that keeps watch locally**

Alternative Agent lines:
- Keep watch at the edge
- Your field signal for production
- A quiet beacon for every app

## UI guidance
- prefer dark mode as the hero treatment
- use soft glows sparingly around active states
- charts should use cyan, sky, amber, and rose over deep dark surfaces
- avoid loud gradients and glossy effects
- use the beacon mark in integration and source-management surfaces
- use Lantern D for all app-level identity moments

## Do and do not

### Do
- keep the mark geometric and simple
- preserve strong negative space
- design favicon-first
- keep dark and light variants
- use the tower as a supporting sub-brand only

### Do not
- no eye symbol
- no shield as the main mark
- no fantasy character or hooded figure
- no medieval details
- no generic line-chart logo
- no neon overload

## Implementation suggestion
- main app shell uses Lantern D
- agent install pages use Beacon Tower in cards and badges
- source status cards can use tiny tower glyphs
- trace and anomaly highlights can echo the Lantern D beam shape
